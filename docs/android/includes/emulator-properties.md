|属性|描述|选项|
|--- |--- |--- |
|`abi.type`|**ABI 类型**&ndash;指定仿真设备的 ABI （应用程序二进制接口） 类型。 **X86**选项适用于的指令集通常被称为"x86"或"IA-32。" **X86_64**选项适用于 64 位 x86 指令集。 **Armeabi-v7a**选项适用于 ARM 指令集具有 v7-a ARM 扩展。 **Arm64-v8a**选项适用于支持 AArch64 的 ARM 指令集。|x86、x86_64、armeabi-v7a、arm64-v8a|
|`disk.cachePartition`|**缓存分区**&ndash;确定仿真的设备是否将使用 **/cache**在设备上的分区。 **/Cache**分区 （这是最初为空） 是 Android 存储经常访问的数据和应用程序组件的位置。 如果设置为**没有**，将不会使用仿真程序 **/cache**分区，且其他`disk.cache`设置将被忽略。|yes、no|
|`disk.cachePartition.path`|**缓存分区路径**&ndash;指定在开发计算机上的缓存分区映像文件。 在仿真程序将使用此文件以进行 **/cache**分区。 输入绝对路径或相对于仿真程序的数据目录的路径。 如果未设置，仿真器将创建名为的空临时文件**cache.img**在开发计算机上。 如果文件不存在，它则会被创建为一个空文件。 如果忽略此选项`disk.cachePartition`设置为**没有**。||
|`disk.cachePartition.size`|**缓存分区大小**&ndash;缓存分区文件 （以字节为单位） 的大小。 通常不需要设置此选项，除非应用将下载非常大的文件，这些文件大于 66 MB 的默认缓存大小。 如果忽略此选项`disk.cachePartition`设置为**没有**。 如果此值是一个整数，它以字节为单位指定的大小。 您还可以指定大小以千字节、 兆字节为单位和千兆字节为单位通过追加**K**， **M**，或**G**的值。 最小大小是**9 分钟**最大大小为**1023 G**。||
|`disk.dataPartition.initPath`|**数据分区的初始路径**&ndash;指定的数据分区的初始内容。 擦除用户数据之后, 在仿真程序将指定文件的内容复制到用户数据 (默认情况下**userdata qemu.img**) 而不是使用**userdata.img**作为初始版本。||
|`disk.dataPartition.path`|**数据分区的路径**&ndash;指定用户数据分区文件。 若要配置永久性用户数据文件，请在开发计算机上输入文件名和路径。 如果文件不存在，仿真程序将从默认文件创建映像**userdata.img**，将其存储在由指定的文件名`disk.dataPartition.path`，并在仿真器关闭时，会保留用户数据。 如果不指定路径，默认文件名为**userdata qemu.img**。 特殊值**<temp>** 会使仿真器创建并使用临时文件。 如果`disk.dataPartition.initPath`，则会将其内容复制到`disk.dataPartition.path`在启动时的文件。 注意，此选项不能留空。||
|`disk.dataPartition.size`|**数据分区大小**&ndash;以字节为单位指定用户数据分区的大小。 如果此值是一个整数，它以字节为单位指定的大小。 您还可以指定大小以千字节、 兆字节为单位和千兆字节为单位通过追加**K**， **M**，或**G**的值。 最小大小是**9 分钟**最大大小为**1023 G**。||
|`disk.ramdisk.path`|**Ramdisk 路径**&ndash;到启动分区 (ramdisk) 映像的路径。 ramdisk 映像是装载系统映像之前由内核加载的系统映像的子集。 ramdisk 映像通常包含启动时的二进制文件和初始化脚本。 如果未指定此选项，默认值是**ramdisk.img** ，仿真器系统目录中。||
|`disk.snapStorage.path`|**快照存储路径**&ndash;存储所有快照的快照存储文件的路径。 在执行期间进行的所有快照将被保存到此文件。 只有被保存到此文件的快照在仿真器运行期间才能被还原。 如果未指定此选项，默认值为 snapshots.img 仿真器数据目录中。||
|`disk.systemPartition.initPath`|**系统分区初始路径**&ndash;系统图像文件的只读副本路径; 具体而言，包含系统库和 API 级别和任何变体相对应的数据的分区。 如果未指定此路径，默认值为 system.img，仿真器系统目录中。||
|`disk.systemPartition.path`|**系统分区路径**&ndash;读/写系统分区映像的路径。 如果未设置此路径，将创建临时文件并在从指定的文件的内容初始化`disk.systemPartition.initPath`。||
|`disk.systemPartition.size`|**系统分区大小**&ndash;系统分区 （以字节为单位） 的理想大小。 如果实际的系统分区映像大于此设置，则会忽略此大小；否则，它指定系统分区文件可以增长到的最大大小。 如果此值是一个整数，它以字节为单位指定的大小。 您还可以指定大小以千字节、 兆字节为单位和千兆字节为单位通过追加**K**， **M**，或**G**的值。 最小大小是**9 分钟**最大大小为**1023 G**。||
|`hw.accelerometer`|**加速计**&ndash;确定仿真的设备是否包含一个加速度传感器。 加速计帮助设备确定方向（用于自动旋转）。 加速计报告设备沿着三个传感器轴的加速情况。|yes、no|
|`hw.audioInput`|**音频录制支持**&ndash;确定仿真的设备是否可以录制音频。|yes、no|
|`hw.audioOutput`|**音频播放支持**&ndash;确定仿真的设备是否可以播放音频。|yes、no|
|`hw.battery`|**电池支持**&ndash;确定仿真的设备是否可以在使用电池运行。|yes、no|
|`hw.camera`|**照相机支持**&ndash;确定仿真的设备是否具有照相机。|yes、no|
|`hw.camera.back`|**后置照相机**&ndash;配置后置照相机 （镜头背朝用户）。 如果在开发计算机上使用网络摄像头以模拟仿真设备上的后置照相机，此值必须设置为网络摄像机*n*，其中_n_选择网络摄像头 （如果只有一个网络摄像头选择**webcam0**)。 如果设置为模拟，仿真程序模拟软件中的照相机。 若要禁用后置照相机，请将此值设置为无。 如果启用后置照相机，请确保也要启用`hw.camera`。|emulated、none、webcam0|
|`hw.camera.front`|**前置相机**&ndash;配置前置照相机 （镜头面向用户）。 如果在开发计算机上使用网络摄像头以模拟仿真设备上的前置照相机，此值必须设置为网络摄像机*n*，其中_n_选择网络摄像头 （如果只有一个网络摄像头选择**webcam0**)。 如果设置为模拟，仿真器将模拟摄像机软件中。 若要禁用前置照相机，请将此值设置为无。 如果启用前置照相机，请确保也要启用`hw.camera`。|emulated、none、webcam0|
|`hw.camera.maxHorizontalPixels`|**最大水平像素**&ndash;配置仿真的设备照相机 （以像素为单位） 的最大水平分辨率。||
|`hw.camera.maxVerticalPixels`|**最大垂直像素**&ndash;配置仿真的设备照相机 （以像素为单位） 的最大垂直分辨率。||
|`hw.cpu.arch`|**CPU 体系结构**&ndash;的 CPU 体系结构，以通过虚拟设备来模拟。 如果使用 Intel HAXM 来执行硬件加速，请选择**x86** 32 位 cpu。 选择**x86_64**的 64 位 HAXM 加速的设备。 (请务必安装 SDK 管理器中的相应 Intel x86 系统映像： 例如，Intel x86 Atom 或 Intel x86 Atom_64。)若要模拟 ARM CPU，请选择**arm**的 32 位或 select **arm64** 64 位 ARM cpu。 请记住，基于 ARM 的虚拟设备运行速度要比基于 x86 的那些虚拟设备慢得多，因为硬件加速不适用于 ARM。|x86、x86_64、arm、arm64|
|`hw.cpu.model`|**CPU 型号**&ndash;此值通常处于取消设置 (它将设置为一个值，派生自`hw.cpu.arch`如果未显式设置)。 但是，可以将它设置为特定于仿真器的字符串以作实验性使用。||
|`hw.dPad`|**DPad 密钥**&ndash;确定仿真的设备是否支持方向键 (DPad) 密钥。 一个 DPad 通常具有四个密钥以指示方向控件。|yes、no|
|`hw.gps`|**支持 GPS** &ndash;确定仿真的设备是否具有 GPS （全球定位系统） 接收器。|yes、no|
|`hw.gpu.enabled`|**GPU 仿真**&ndash;确定仿真的设备是否支持 GPU 仿真。 启用后，GPU 仿真会使用 Open GL for Embedded Systems 以在屏幕上呈现 2D 和 3D 图形，并且关联的 GPU 仿真模式设置会确定 GPU 仿真的实现方式。|yes、no|
|`hw.gpu.mode`|**GPU 仿真模式**&ndash;确定仿真器实现 GPU 仿真的方式。 如果选择自动，在仿真程序将选择根据你的开发计算机设置的硬件和软件加速。 如果你选择的主机，模拟器将使用开发计算机的图形处理器执行 GPU 仿真以进行更快的呈现。 如果您的 GPU 与仿真器兼容并不在 Windows 上，可以尝试而不是主机的角度。 角度模式使用 DirectX 来提供类似于主机的性能。 如果选择 mesa，模拟器将使用 Mesa 3D 软件库来呈现图形。 如果通过开发计算机的图形处理器进行呈现存在问题，请选择 mesa。 Swiftshader 模式可用于呈现具有比使用计算机的 GPU 略有更低性能的软件中的图形。 关闭选项 （禁用图形硬件仿真） 是一种不推荐使用的选项，可能会导致无法正确呈现某些项，因此不推荐使用。|auto、host、mesa、angle、swiftshader、off|
|`hw.gsmModem`|**GSM 调制解调器支持**&ndash;确定仿真的设备是否包含支持 GSM （全球移动通信系统） 电话无线电系统的调制解调器。|yes、no|
|`hw.initialOrientation`|**初始屏幕方向**&ndash;配置屏幕的初始方向 （纵向或横向模式） 在模拟设备上。 在纵向模式下，屏幕的高度大于宽度。 在横向模式下，屏幕的宽度大于高度。 如果设备配置文件中都支持纵向和横向模式，则可以在运行仿真设备时更改方向。|portrait、landscape|
|`hw.keyboard`|**键盘支持**&ndash;确定仿真的设备是否支持全键盘。|yes、no|
|`hw.keyboard.charmap`|**键盘字符映射名称**&ndash;此设备的硬件字符映射的名称。 注意： 此应始终为默认值**qwerty2**除非相应地修改的系统映像。 此名称会在启动时发送到内核。 使用不正确的名称将导致虚拟设备不可用。||
|`hw.keyboard.lid`|**键盘盖支持**&ndash;如果启用键盘支持，则此设置确定是否 QWERTY 键盘可以关闭/隐藏或打开/显示。 如果 hw.keyboard 设置为 false，则将忽略此设置。 注意： 默认值为 false 如果仿真的设备面向 API 级别 12 或更高版本。|yes、no|
|`hw.lcd.backlight`|**LCD 背光**&ndash;确定 LCD 背光是否由仿真设备模拟。|yes、no|
|`hw.lcd.density`|**LCD 密度**&ndash;密度仿真 LCD 显示，以密度无关的像素或 dp （dp 是一个虚拟像素单位）。 当设置为 160 dp 时，每个 dp 将对应一个物理像素。 在运行时，Android 使用此值选择和缩放适当的资源/资产以进行正确的显示呈现。|120、160、240、213、320|
|`hw.lcd.depth`|**LCD 颜色深度**&ndash;颜色位深度保留位图以驱动 LCD 显示的仿真帧缓冲区。 此值可以为 16 位（65,536 种可能的颜色）或 32 位（16,777,216 种颜色和透明度）。 尽管 32 位设置使仿真器运行较为缓慢，但颜色准确度更高。|16, 32|
|`hw.lcd.height`|**LCD 像素高度**&ndash;构成仿真 LCD 显示的垂直维度的像素数。||
|`hw.lcd.width`|**LCD 像素宽度**&ndash;构成仿真 LCD 显示的水平维度的像素数。||
|`hw.mainKeys`|**硬件返回/主页键**&ndash;确定仿真的设备是否支持硬件返回和主页导航按钮。 可以将此值设置为**是**如果仅在软件中实现按钮。 如果`hw.mainKeys`设置为**是**、 仿真程序不会显示导航按钮在屏幕上，但可以使用仿真器侧面板来"按"这些按钮。|yes、no|
|`hw.ramSize`|**设备 RAM 大小**&ndash;仿真设备，用兆字节表示的物理 RAM 量。 默认值将根据屏幕大小或外观版本来计算。 尽管增加大小可以提供更快的仿真器操作，但这将耗费开发计算机中更多的资源。||
|`hw.screen`|**触摸屏幕上，键入**&ndash;仿真设备上定义的屏幕的类型。 多点触控屏幕可以在触控界面上跟踪两个或多个手指。 触摸屏可以检测到只有单个手指触控事件。 无触摸屏不会检测触控事件。|touch、multi-touch、no-touch|
|`hw.sdCard`|**SDCard 支持**&ndash;确定仿真的设备是否支持插入和移除虚拟 SD （数字安全） 卡。 仿真程序使用存储在开发计算机上可装载的磁盘映像来模拟真实 SD 卡设备 （请参阅 hw.sdCard.path） 的分区。|yes、no|
|`sdcard.size`|**SDCard 大小**&ndash;指定指定的位置处的虚拟 SD 卡文件的大小`hw.sdCard.path`。 （以字节为单位） 设备上可用。 如果此值是一个整数，它以字节为单位指定的大小。 您还可以指定大小以千字节、 兆字节为单位和千兆字节为单位通过追加**K**， **M**，或**G**的值。 最小大小是**9 分钟**最大大小为**1023 G**。||
|`hw.sdCard.path`|**SDCard 映像路径**&ndash;指定文件名和路径在开发计算机上 SD 卡分区映像文件。 例如，此路径可能设置为**C:\sd\sdcard.img**在 Windows 上。||
